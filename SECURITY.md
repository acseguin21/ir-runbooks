# Security

## Reporting a vulnerability

If you find a security issue in this repository (e.g. unsafe guidance, dependency issue), please report it responsibly:

- **Do not** open a public issue for security-sensitive findings.
- Use **GitHub Security Advisories** (repo **Security** tab → **Advisory**) or contact the maintainer privately.
- Include description and steps to reproduce; we will respond and coordinate disclosure.

## Scope

- This repo contains **generic runbook and playbook templates only**. No secrets, internal procedures, or customer data belong here.
- Adapt all content to your environment; do not paste internal hostnames, credentials, or IOCs.

## Best practices (this repo)

- No credentials, API keys, or connection strings.
- No real IOCs, internal asset names, or customer identifiers.
- Runbooks are templates; validate and approve through your org’s process before use.
